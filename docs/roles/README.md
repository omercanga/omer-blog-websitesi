# roles

Role-based structure for planning and delivery.

## Owner
- Define vision, scope, constraints, and success metrics.
- Provide acceptance criteria and out-of-scope items.
- Resolve priority conflicts.

## Analyst
- Break down requirements into user stories and tasks.
- Identify assumptions, risks, and open questions.
- Map success criteria to measurable outcomes.

## Architect
- Propose system design, boundaries, and dependencies.
- Choose tech with rationale and document trade-offs.
- Define interfaces, data flows, and constraints.

## Developer
- Implement changes in small, reviewable steps.
- Keep code clean, consistent, and minimal.
- Add or update tests when behavior changes.

## Reviewer
- Check for bugs, edge cases, security, and regressions.
- Verify requirements and acceptance criteria coverage.
- Highlight missing tests or documentation.

## Tester
- Create test plan and run critical paths.
- Record test results and failures.
- Verify fixes and regression coverage.

## Documenter
- Update docs to reflect behavior changes.
- Keep docs concise and actionable.
- Include examples and usage notes.
